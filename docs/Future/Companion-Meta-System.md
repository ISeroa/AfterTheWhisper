#Companion Meta Systems (Future)

##Overview

동료가 단순 전투 유닛이 아니라 "사람"으로 느껴지도록 만드는 메타/서사 시스템 아이디어 모음.
현재 구현 스코프 밖의 기능들을 Future로 분리해, 우선순위를 흔들지 않으면서도 확장 방향을 보관한다.

##Key Decisions

-구현 부담이 큰 시스템은 Future로 격리: 핵심 전투 루프/AI 안정화에 집중.

-감정/책임을 만드는 장치 우선: 리더로서의 선택과 결과가 남도록 설계.

-세계관 일관성 유지: 세상은 크게 반응하지 않지만, 기록은 남는다.

##Architecture

-(저비용 확장 기준) SaveGame 기록 + UI 위젯 + 간단 상호작용 액터로 구성

-(AI와 분리) 전투 AI는 그대로 두고, 메타 시스템은 GameInstance/SaveGame 레벨에서 관리

-(이벤트 기반) 사망/다운/임무결과 같은 이벤트를 수신해 기록 갱신

##Trade-offs

-메타 시스템을 넣으면 UI/데이터/세이브 공수가 증가

-감정적 무게가 커져 난이도/피로도 조절이 필요

##Future

###1) Down → Rescue → Permanent Loss

-동료 HP 0 시 즉사 대신 Down 상태 진입

-제한 시간 내 구조 시 생존

-실패 시 영구 사망(해당 동료는 세계관에서 재등장 불가)

###2) Player Death → Base Return + Squad Extraction Roll

-플레이어 사망 시 베이스로 복귀

-동료는 귀환/부상/실종/사망 중 하나로 판정(확률은 동료 상태/상황에 의해 결정)

###3) Memorial Space (Base Interaction + Paged UI)

-거점에 고정된 추모 공간(월드 오브젝트)

-상호작용 시 UI로 기록 목록 표시

-스크롤 대신 페이지 방식(찾기/정렬 용이)

-큰 연출 변화 없음(톤 유지), 가끔 짧은 한두마디 정도의 가벼운 대사만

###4) Expedition / Delegation Events

-플레이어가 직접 가지 않고 동료만 파견 가능한 이벤트

-동료 스탯/성격에 따라 성공률/결과 변화

-결과: 성공/부상/납치/실종 등

-플레이어가 직접 가면 확률/결과가 개선

###5) Growth System (Optional)

-동료 경험치/숙련도/신뢰도 등 성장 축

-전투 성능뿐 아니라 행동 성향(후퇴/집결/자가치료 판단)에도 영향

###6) Ritual Recovery (Very Optional)

-영구 사망을 되돌리는 "특수 의식" 아이디어

-큰 희생/리소스 소모를 전제(세계관적 대가)

-기본은 영구 사망 유지, 예외 규칙으로만 존재
